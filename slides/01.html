Python 2 vs Python 3

What are the differences?

Which one should you start with?

Python 3 is a backward-incompatible new version. Under development since May 2006 (under the code name Python 3000). Most of the language is the same, but there are key differences that make it incompatible.

  • Module reorganization
  • Lists are replaced with iterators as return values for most standard functions
  • Expanded/more stringent Unicode support
  • New syntax features
  • Small number of syntax features removed

Simple decision

  • If you have no external module dependencies, use 3
  • If all of the libraries you depend work with 3, use 3
  • If you can port or drop libraries that only support 2, use 3
  • Otherwise, use 2.7 and write careful, good code that requires few changes to go to 3

For what it’s worth, I use python 2.7

Installing Python
with Homebrew

We all use Mac OS, so install python with homebrew. On servers, install Python with a package manager. If you’re on Windows, cry (but installers are available)

Be sure to install Python 2.7 or Python 3.2 (or later). These versions introduced broad changes to make porting or single codebases easier.

Other (lesser) options

Built-in Python | MacPython

The version of Python included with MacOS is usually a bit old, but is reasonably recent in Mavericks. MacOS doesn't come with Python 3.

MacPython is an old installer-based version of Python. You don’t need it or want it, but you may come across references to it.
